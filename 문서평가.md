**리스크**: 15번 파일은 "SIDE vision fail"(현상)로 저장되어 있으나 명세서는 "Teaching incomplete"(원인)로 부름. 시뮬레이터·파서가 동일 파일을 서로 다른 이벤트로 분류할 수 있음.

### 2.2 페이로드 구조 정합성 — 우수

| 검증 항목 | 결과 |
|---|---|
| 공통 헤더 5필드 | ✅ 전 20파일 준수 (HEARTBEAT만 4필드, spec 일치) |
| inspection_detail PascalCase | ✅ 준수 |
| SIDE Offset 전부 0 (실측 패턴) | ✅ 전수 준수 |
| detail 그룹 존재 | ✅ INSPECTION_RESULT 5종 모두 |
| UUID v4 / ISO 8601 밀리초 | ✅ 전수 통과 |

### 2.3 실측 패턴 보존도 — 매우 우수

- **04 (PASS)**: ZA5 XOffset=121, YOffset=-94 — spec "실측: -25~121" 범위 정확히 일치
- **07 (ET=11)**: ZA0 XOffset=73, ZA4 XOffset=102 — spec "73~102 (FAIL ET=11)"와 일치
- **08 (혼재)**: ZA4~7 ET=12, ZA1~3 ET=52 — Teaching 과도기 패턴 재현

### 2.4 병목 방지 관점 — 1건 최적화 필요

- INSPECTION_RESULT 페이로드 약 2.1KB, takt 1,620ms. 전 슬롯 PASS인데도 detail 그룹을 매 Unit마다 싣는 구조는 낭비.
- **권고**: PASS일 때 detail 그룹 drop, Strip 단위 1회 요약. Mock에 `_drop_hint: true` 메타 추가.
